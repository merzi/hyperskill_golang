Description

In this stage, we will work on the basic functionality of our CLI program. It includes the ability to create, print, or delete all notes.

Objectives
  • Modify the solution for the previous stage by adding three new commands â€“ create, list, and clear.

When the command is create:

  • The program should interpret the data as a type string and make a new note that contains this text. The program should be capable of storing up to 5 notes in its memory. The number of notes is a random number. We guarantee that it will be a relatively small number to provide the readability and test-ability;
  • If the operation was successful, the program should print [OK] The note was successfully created\n;
  • If users try to exceed the maximum number of notes, print [Error] Notepad is full\n.

When the command is clear:

  • The program should delete all records and print [OK] All notes were successfully deleted\n.

When the command is list:

  • The program should print all entries using [Info] %d: %s\n as a format string. It contains the position (as an integer) and the text of a note at this position. The position starts at 1. Don't print empty entries.

The program should run in an infinite loop and terminate only when the exit command is received.

Examples

The greater-than symbol followed by a space (> ) represents the user input. Note that it's not part of the input.

Example 1: program execution

Enter a command and data: > create This is my first record!
[OK] The note was successfully created

Enter a command and data: > list
[Info] 1: This is my first record!

Enter a command and data: > clear
[OK] All notes were successfully deleted

Enter a command and data: > list

Enter a command and data: > exit
[Info] Bye!
